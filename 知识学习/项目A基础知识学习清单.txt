项目A基础知识学习清单

说明：
这份清单用于后续逐个知识点学习。建议不要一次性全部学完，而是围绕项目A的实现流程逐步学习：

数据 -> Prompt -> 多模态模型推理 -> 输出保存 -> 答案解析 -> 指标计算 -> verifier 进阶


一、LLM 基础

1. Transformer decoder
- 什么是 decoder-only Transformer
- masked self-attention 是什么
- 为什么 LLM 可以一个 token 一个 token 生成文本
- Transformer decoder 和 encoder 的区别

2. Token
- token 是什么
- token 和字、词、字符的关系
- tokenizer 的作用
- 为什么中文、英文、公式、LaTeX 会被切成不同 token

3. Context length
- context length 是什么
- prompt、图片 token、历史输出都会占用上下文长度
- context length 太短会发生什么
- 为什么长 CoT 会增加上下文压力

4. Autoregressive generation
- 自回归生成是什么意思
- 为什么模型每次只预测下一个 token
- prefill 和 decode/generation 阶段的区别
- 生成过程为什么不能完全并行

5. Temperature
- temperature 控制什么
- temperature=0 和 temperature>0 的区别
- 为什么评测时常用低 temperature
- temperature 对答案稳定性和多样性的影响

6. Top-p
- top-p / nucleus sampling 是什么
- top-p 和 temperature 的区别
- top-p 如何影响生成多样性
- best-of-N 中为什么可能需要采样

7. Max tokens / max_new_tokens
- max tokens 是什么
- max_new_tokens 和 context length 的关系
- 设置太小会导致答案截断
- 设置太大会增加耗时和显存压力

8. Prompt engineering
- prompt 的作用
- system prompt、user prompt 的区别
- 如何要求模型输出固定格式
- 如何减少模型不按格式输出的问题

9. Chain-of-Thought
- CoT 是什么
- 为什么复杂推理任务会要求 step-by-step
- CoT 对答案质量和评测解析的影响
- CoT 输出可能带来的幻觉和冗余问题


二、多模态大模型 MLLM 基础

1. MLLM 是什么
- 多模态大模型和普通 LLM 的区别
- 图像输入如何参与语言生成
- MLLM 的典型任务：VQA、图像描述、OCR、图表理解、数学视觉推理

2. 图像编码器 ViT
- Vision Transformer 是什么
- image patch 是什么
- ViT 如何把图片变成一组向量
- ViT 和 CNN 的直观区别

3. CLIP
- CLIP 是什么
- 图像-文本对比学习的基本思想
- 为什么 CLIP 常被用作 vision encoder
- CLIP 的优势和局限

4. Vision encoder
- vision encoder 的输入和输出是什么
- 图像特征和文本 token embedding 有什么不同
- 为什么视觉编码器通常先预训练好

5. Projector
- projector 是什么
- 为什么需要把视觉特征映射到 LLM embedding 空间
- 线性 projector、MLP projector、Q-Former 的区别
- LLaVA 中 projector 的作用

6. LLM decoder
- MLLM 中的语言模型负责什么
- LLM 如何结合 visual tokens 和文本 prompt 生成回答
- 为什么最终输出仍然是文本 token

7. Vision encoder + projector + LLM decoder 结构
- LLaVA 的基本结构
- Qwen2.5-VL 的基本结构
- visual tokens 如何进入语言模型上下文
- 这个结构对项目A推理有什么意义

8. Image token
- image token 是什么
- 一张图片为什么会占用很多 token
- image token 数量和图片分辨率的关系
- 多图输入对 context length 的影响

9. 动态分辨率
- 动态分辨率是什么意思
- 为什么高分辨率图像会带来更多 visual tokens
- Qwen2.5-VL 动态分辨率的大致思想
- 数学图、几何图、表格图为什么需要较高分辨率

10. OCR 理解
- OCR 是什么
- MLLM 如何读取图片里的文字
- OCR 错误对数学题和图表题的影响
- 如何在 bad case 中分析 OCR 失败

11. 图表理解
- 图表题包含哪些视觉信息
- 坐标轴、图例、刻度、趋势如何影响答案
- MLLM 在图表理解中常见错误

12. 几何题理解
- 几何图像中的点、线、角、长度关系
- 视觉识别和数学推理如何结合
- 几何题中常见的视觉误判

五、过程评估 / Verifier 进阶

1. Verifier 是什么
- verifier 的作用
- verifier 和 policy model 的区别
- 为什么 verifier 可以帮助提高最终准确率

2. 判断 CoT 中间步骤是否合理
- 什么是过程评估
- step-level correctness 是什么
- 为什么真正的 process reward model 需要步骤级标签
- 没有步骤标签时该怎么办

3. Best-of-N
- best-of-N 是什么
- 为什么同一道题生成多个答案可能更好
- N 如何选择
- best-of-N 的计算成本

4. 用 verifier 选择答案
- 对每道题生成多个候选 CoT
- verifier 给每个候选打分
- 选择分数最高的候选作为最终答案
- 和 majority vote 的区别

5. LLM-as-judge
- LLM-as-judge 是什么
- 用强模型评价弱模型输出的基本方法
- LLM-as-judge 的优点和风险
- 为什么项目A基础评测不应完全依赖 LLM-as-judge

6. Reward model
- reward model 是什么
- outcome reward model 和 process reward model 的区别
- 如何用最终答案正确性训练 outcome verifier

7. Process reward model
- PRM 是什么
- PRM 为什么关注中间推理步骤
- PRM 需要什么训练数据
- 为什么项目A中 PRM 属于进阶选做

8. 项目A中推荐的 verifier 路线
- 第一阶段：不训练 verifier，先完成 baseline
- 第二阶段：用模型多次采样构造候选答案
- 第三阶段：用手写评测结果给候选打 0/1 标签
- 第四阶段：训练轻量 answer-level verifier
- 第五阶段：比较 baseline、best-of-N、verifier rerank 的 Accuracy
